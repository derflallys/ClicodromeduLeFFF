L'import du lexique prend en charge deux syntaxes de fichiers différentes présentées ci-contre :
Syntaxe 1 (.txt)
Cette syntaxe correspond à celle respectée par notre fonctionnalité d'export : 
                  <mot> <code-categorie>["<categorie-name>"] lemme="<mot>" {<mot-tags>}
                  <forme-fléchie-générée> f["forme fléchie"] lemme="<mot>" {<tags-combination>}
                  <mot> <code-categorie>["<categorie-name>"] lemme="<mot>" {<mot-tags>}
                  <forme-fléchie-générée> f["forme fléchie"] lemme="<mot>" {<tags-combination>}
                  <forme-fléchie-générée> f["forme fléchie"] lemme="<mot>" {<tags-combination>}
                  ...
⚠️Attention chaque espacement doit être une tabulation. Elles sont au nombre de 3. ⚠️
<mot> : le mot de la base
<code-categorie> : le code de la catégorie du mot (ex: "adj" est le code de "adjectif")
<categorie-name> : le nom complet de la catégorie
<mot-tags> : les tags renseignés sur le mot. Chaque tag est séparé par un ";"
<forme-fléchie-générée> : une forme fléchie du mot présenté précédemment
f["forme fléchie"] : "f" correspond au code de la catégorie "forme fléchie"
lemme="<mot>" : avec <mot> correspondant au lemme de base de la forme fléchie
<tags-combination> : l'association de tags ayant permis de générer la forme fléchie. Chaque tag est séparé par un ";"
 Syntaxe 2 (.txt)
Cette syntaxe correspond à la syntaxe du Lefff consultable le site de Lionel CLEMENT.

⚠️Attention ce format ne contient pas toutes les informations nécéssaires au bon fonctionnement de l'application. En effet cet import permet d'ajouter les mots en base de données, ainsi que leurs catégories.
Néamoins les associations de tags de chaque catégorie, utilisées par les règles PFM pour générer les formes fléchies ne sont pas présentes. Il vous faudra donc passer par l'administration pour les ajouter vous même.
Les tags concernant les mots existent mais restent très peu enrichis. (ex: pas de tag pour indiquer le groupe des verbes ect...)

Le format est le suivant : 
                  <mot> <code-categorie> <lemme> <tags-mot>
                  <mot> <code-categorie> <lemme> <tags-mot>
                  <mot> <code-categorie> <lemme> <tags-mot>
                  ...
⚠️Attention chaque espacement doit être une tabulation.⚠️
<mot> : une entrée du lexique
<code-categorie> : le code de la catégorie du mot (ex: "adj" est le code de "adjectif")
<lemme> : correspond au lemme de base du mot. Si le mot est une forme fléchie d'un autre mot alors <mot> != <lemme> sinon <mot> == <lemme>
<tags-mot> : les tags renseignés sur le mot. Chaque tag est représenté par un caractère unique. (ex: "Y1p" = [Y]impératif(présent) [1]1ère personne [p]pluriel)

L'export du lexique permettra d'extraire tous les mots enregistrés en base de données dans l'ordre alphabétique. Les formes fléchies de chaque mot seront générées et presentées dans l'export à la suite du mot correspondant en respectant le format ci-contre : 
            <mot> <code-categorie>["<categorie-name>"] lemme="<mot>" {<mot-tags>}
            <forme-fléchie-générée> f["forme fléchie"] lemme="<mot>" {<tags-combination>}
            <mot> <code-categorie>["<categorie-name>"] lemme="<mot>" {<mot-tags>}
            <forme-fléchie-générée> f["forme fléchie"] lemme="<mot>" {<tags-combination>}
            <forme-fléchie-générée> f["forme fléchie"] lemme="<mot>" {<tags-combination>}
            ...
⚠️Attention chaque espacement est en réalité une tabulation ⚠️
<mot> : le mot de la base
<code-categorie> : le code de la catégorie du mot (ex: "adj" est le code de "adjectif")
<categorie-name> : le nom complet de la catégorie
<mot-tags> : les tags renseignés sur le mot. Chaque tag est séparé par un ";"
<forme-fléchie-générée> : une forme fléchie du mot présentés précédemment
f["forme fléchie"] : "f" correspond au code de la forme fléchies
lemme="<mot>" : avec <mot> correspondant au lemme de base de la forme fléchie
<tags-combination> : l'association de tags ayant permis de générer la forme fléchie. Chaque tag est séparé par un ";"
